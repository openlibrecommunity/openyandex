Likely search strategy
1. Use yandex-pdb-index.jsonl as the source of truth for all symbol probes.
2. Probe candidate URLs with HEAD and ranged GET, because some symbol servers do not support HEAD correctly.
3. Search payload strings for symbol server, crash upload, update, and debug endpoints.
4. Search GitHub and public web for exact GUIDAge strings and PDB names.
5. Check Yandex crashpad endpoints and symbol upload/download conventions.
6. Try both PDB and compressed pd_ forms.
7. Try case variants for PDB names only if endpoint behavior suggests case sensitivity.

Why this matters
- If full browser.dll.pdb is found, the project changes from months of hard reconstruction to a much more tractable source mapping and patch extraction job.
- Even private module PDBs like cspeechkit/textclassifier would significantly improve naming and subsystem boundaries.
